## About

This repository documents a hands-on learning journey through frontend development. Each folder contains a standalone mini-project that explores a specific CSS technique or JavaScript interaction. There are no build tools, no frameworks, and no dependencies. Open any `index.html` file directly in a browser and it works.

The goal is to build a strong foundation in the core technologies of the web before moving on to frameworks.

## Getting Started

### Prerequisites

You need only two things:

- A modern browser (Chrome, Firefox, Edge, Safari)
- A code editor (VS Code recommended)

No Node.js, no Python, no installations required.

### Installation

1. Clone the repository:

```bash
git clone https://github.com/AndreiIliescu/self-learning-frontend-development.git
```

2. Navigate into the project folder:

```bash
cd self-learning-frontend-development
```

3. Open any project directly in your browser:

```bash
cd dark_mode_switch
open index.html        # macOS
start index.html       # Windows
xdg-open index.html    # Linux
```

Or open the file manually from your file explorer.

If you use VS Code, install the [Live Server](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer) extension and click "Go Live" for automatic reload on save.

## Contributing

Contributions are welcome. If you want to suggest an improvement or add a new mini-project, follow these steps:

1. Fork the repository.
2. Create a new branch:

```bash
git checkout -b feature/your-feature-name
```

3. Make your changes and commit:

```bash
git commit -m "Add: brief description of what you added"
```

4. Push to your fork:

```bash
git push origin feature/your-feature-name
```

5. Open a Pull Request on GitHub.

Each new project must follow the existing folder structure: one folder per project, with its own `index.html`, `style.css`, and `script.js` (if JavaScript is needed). No external libraries or CDN links.
